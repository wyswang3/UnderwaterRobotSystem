## 1.2) 我们下一步要打磨的“命令行细节”是什么（基于你现状）

你目前的 CLI 已经可用，但调试时你一定会遇到两个痛点：

1. **全量画图太慢、太大**（尤其 IMU 高频）
2. **只想看某个时间窗**（例如下水前 30s 或某段动作）

因此我建议我们为 `plot-raw` 增加非常轻量的参数，不破坏现有用法：

* `--t0` / `--t1`：只绘制时间窗（相对 run 的 t_s，单位秒）
* `--max-points` 或 `--downsample`：绘图点数上限/降采样
* `--format png|pdf`：论文导出用 pdf（可选）

保持默认行为不变：不传这些参数就画全量并输出 png。

只看下水前 60 秒（假设时间是 EstS/MonoS 的“绝对秒”）
(offnav_env) python -m offnav.cli_raw --dataset-config ../configs/dataset.yaml \
  plot-raw --run 2026-01-10_pooltest01 --t0 0 --t1 60

限制 IMU 绘图最多 5000 点（全程画，但不会卡死）
(offnav_env) python -m offnav.cli_raw --dataset-config ../configs/dataset.yaml \
  plot-raw --run 2026-01-10_pooltest01 --max-points 5000

只画 DVL（调试 DVL 时更快）

(offnav_env) python -m offnav.cli_raw --dataset-config ../configs/dataset.yaml \
  plot-raw --run 2026-01-10_pooltest01 --no-imu

## 最传统实现：
python -m offnav.cli_raw --dataset-config ../configs/dataset.yaml plot-raw --run 2026-01-10_pooltest01 --out-dir ../out/plots_raw
python -m offnav.cli_raw --dataset-config ../configs/dataset.yaml plot-raw --run 2026-01-10_pooltest02 --out-dir ../out/plots_raw
python -m offnav.cli_raw --dataset-config ../configs/dataset.yaml plot-raw --run 2026-01-10_pooltest03 --out-dir ../out/plots_raw

## 2. cli_proc.py：IMU / DVL 预处理 + 滤波 + 落盘

这是我们正式在用的预处理入口。

### 2.1 只预处理 IMU

输出：

* `../out/proc/2026-01-10_pooltest01/2026-01-10_pooltest01_imu_filtered.csv`
* 对应的 IMU 滤波图

命令：

```bash
(offnav_env) python -m offnav.cli_proc \
  --dataset-config ../configs/dataset.yaml \
  preprocess-imu \
  --run 2026-01-10_pooltest01 \
  --out-dir ../out/proc
```

### 2.2 只预处理 DVL

输出：

* `..._dvl_filtered_all.csv`
* `..._dvl_filtered_BI.csv`
* `..._dvl_filtered_BE.csv`
* 以及 BI 子集的滤波速度图

命令：

```bash
(offnav_env) python -m offnav.cli_proc \
  --dataset-config ../configs/dataset.yaml \
  preprocess-dvl \
  --run 2026-01-10_pooltest01 \
  --out-dir ../out/proc
```

### 2.3 一键预处理 IMU + DVL（推荐）

这是后续 ESKF 管线所依赖的“标准步骤”。

```bash
(offnav_env) python -m offnav.cli_proc \
  --dataset-config ../configs/dataset.yaml \
  preprocess-all \
  --run 2026-01-10_pooltest01 \
  --out-dir ../out/proc \
  --imu-sensor-to-body-map rotz_p90 \
  --imu-mount-rpy-deg 0,0,0
```

预处理完成后，对应 run 的目录结构大致是：

```text
../out/proc/
└── 2026-01-10_pooltest01
    ├── 2026-01-10_pooltest01_imu_filtered.csv
    ├── 2026-01-10_pooltest01_dvl_filtered_all.csv
    ├── 2026-01-10_pooltest01_dvl_filtered_BI.csv
    ├── 2026-01-10_pooltest01_dvl_filtered_BE.csv
    └── 若干 PNG 图
```

## 3. cli_nav.py：轨迹计算（死算 + ESKF）

### 3.1 死算轨迹（Dead-reckon）

直接基于 IMU +（可选）DVL 做简单 deadreckon，结果保存为 Trajectory + 3D 图。

```bash
(offnav_env) python -m offnav.cli_nav \
  --dataset-config ../configs/dataset.yaml \
  --nav-config     ../configs/nav.yaml \
  deadreckon \
  --run     2026-01-10_pooltest01 \
  --out-dir ../out/nav_deadreckon
```

典型输出（终端）会类似：

```text
[DEADRECKON] n_imu=... n_dvl=...
[DEADRECKON] Trajectory saved to: ../out/nav_deadreckon/...
[DEADRECKON] Figure saved to:     ../out/nav_deadreckon/..._traj_3d_deadreckon.png
```

### 3.2 ESKF 轨迹（基于预处理后的 IMU + DVL）

注意：ESKF 必须依赖第 2 步的“预处理 CSV”，所以要先跑 `preprocess-all`。

```bash
(offnav_env) python -m offnav.cli_nav \
  --dataset-config ../configs/dataset.yaml \
  --nav-config     ../configs/nav.yaml \
  eskf \
  --run      2026-01-10_pooltest01 \
  --proc-dir ../out/proc \
  --out-dir  ../out/nav_eskf
```

这里：

* `--proc-dir ../out/proc`：里面会自动去找
  `../out/proc/2026-01-10_pooltest01/2026-01-10_pooltest01_imu_filtered.csv`
  和 DVL 的三个 `*_dvl_filtered_*.csv`。
* 输出包括：

  * ESKF 轨迹 CSV（Trajectory）
  * 3D ENU 轨迹图（现在已经是“机器人视角”的浅绿轨迹 + 蓝背景）

终端日志大致会包含：

```text
[ESKF] n_imu=...  n_dvl_all=...  BI_used=...  BE_used=...  yaw_used=...
[ESKF] Trajectory saved to: ../out/nav_eskf/...
[ESKF] Figure saved to:     ../out/nav_eskf/..._traj_3d_eskf.png
```
